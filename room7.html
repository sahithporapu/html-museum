Semantic tags are HTML5 elements that clearly describe the meaning of the content they contain. These tags tell both the browser and the developer what kind of content is inside, making code more readable and accessible.

Before HTML5, developers used non-semantic tags like <div> and <span> for everything, often with classes or IDs to describe the purpose. Semantic tags solve this by providing meaningful names for common parts of a webpage.

âœ¨ Why Use Semantic Tags?

  • They improve the structure and clarity of your HTML.
  • They help search engines better understand your page (SEO).
  • They make your site more accessible to screen readers.
  • They replace generic tags like <div> with meaningful elements.

ðŸ”– Common Semantic Tags

Tag Description
<header> Defines the top part of a page or section (like a title or logo)
<nav> Contains navigation links (menus)
<main> The main content area of the page
<section> Groups related content together (like rooms in your museum)
<article> Used for independent, reusable content (like a blog post)
<aside> Holds side content like ads, notes, or references
<footer> Defines the bottom area of a page (credits, contact, etc.)
<figure> Wraps images or diagrams with optional captions
<figcaption> Gives a caption to the image inside <figure>
